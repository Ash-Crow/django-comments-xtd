{% trans "To extend django-comments-xtd follow the next steps:" %}

  1. {% trans "The setting COMMENTS_APP must be django_comments_xtd" %}
  2. {% trans "The setting COMMENTS_XTD_MODEL must be the new model class name. For the case of this demo: mycomments.models.MyComment" %}
  3. {% trans "The setting COMMENTS_XTD_FORM_CLASS must be the new form class name. For the case of this demo: mycomments.forms.MyCommentForm" %}
  4. {% trans "Change the following templates:" %}
    1. comments/form.html, to include new fields, and maybe to avoid requesting users to type their name, email and URL whenever they are logged in.
    2. comments/preview.html, to preview the new fields.
    3. django_comments_xtd/email_confirmation_request.{txt|html}, to add the new fields to the confirmation request, if that's necessary (this demo overrides them to include the title field in the email).
    4. django_comments_xtd/comment_tree.html, to add the new field to the list of comments. If your project doesn't allow nested comments, you might want to override comments/list.html instead.
